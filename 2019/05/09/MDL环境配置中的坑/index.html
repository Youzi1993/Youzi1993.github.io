MDL环境配置中的坑

近期，开始学习利用MDL进行Microstation的二次开发。Microstation二次开发有三张，一个是利用VBA进行开发，一个是利用C#进行开发叫Addins，最后一个是利用C++进行开发叫MDL。MDL相对来说开发难度较大，但是能够实现的功能也更多。如果用C#开发，借助Visual Studio和.NET平台，开发环境配置非常方便。但是，如果用C++开发，需要安装Microstation SDK，利用其bmake程序进行编译，中间遇到的不少坑。
参考资料: 一步步学习MDL

1.短名称问题

8.3格式通常指较旧的Windows操作系统或DOS的文件命名限制

8指文件名最大长度为8

3指扩展名最大长度为3

若不符合以上限制则会以”~”作延长名称如”Program Files”会变成”Progra~1”

若同一文件夹有相似的名称，末端的数值则会自动递增

有些操作系统会将短名称功能关闭，这样在编译的时候会出现“不能打开文件:XXXXX的错误”。我最早碰到这个问题的时候，在中国优先社区还没有出现解决方法，阻挡我学习的角度。最近，在社区一步步学习MDL中提供了一个链接：Windows Server 2012 File Server Tip: Disable 8.3 Naming (and strip those short names too)。该链接中提到了windows的一个工具FSUTIL能够开启8.3短名称。

设置是否开启短名称

开启短名称后还会碰到一个问题，就是已有的文件夹还是没有短名称，例如Visual Studio 和Microstation的安装路径。我先把文件夹复制出来，然后把已有的文件夹删掉，再复制回去就能够生成短路径了，或者卸载之后把文件夹删除干净，重新安装应该也能实现。
